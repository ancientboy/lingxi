# 文件预览功能使用说明

## 功能概述

灵犀云现在支持在对话中直接预览和下载文件！

### 支持的文件类型

| 类型 | 扩展名 | 预览方式 |
|------|--------|---------|
| **图片** | .png, .jpg, .jpeg, .gif, .webp, .svg | 直接显示在对话中 |
| **PDF** | .pdf | 新窗口打开预览 |
| **其他** | .doc, .docx, .xls, .xlsx, .ppt, .pptx, .txt, .md, .json | 提供下载 |

---

## 使用方法

### 1. Agent 生成文件

当 Agent 生成文件时，会在消息中包含文件路径：

```
图表已生成：/workspace/chart.png
```

### 2. 前端自动识别

前端会自动识别消息中的文件路径，并显示预览：

```
┌────────────────────────┐
│ Agent: 图表已生成       │
│                        │
│ ┌──────────────────┐   │
│ │  [图片预览]       │   │
│ │  chart.png       │   │
│ │  📥 下载          │   │
│ └──────────────────┘   │
└────────────────────────┘
```

### 3. 用户操作

- **图片**: 点击图片可放大查看
- **PDF**: 点击"预览"按钮在新窗口打开
- **下载**: 点击"下载"按钮保存文件

---

## 技术实现

### 架构

```
用户请求 → Agent 生成文件
              ↓
        保存到 workspace/
              ↓
        文件服务 (端口 9876)
              ↓
        前端识别路径 → 显示预览
```

### 文件服务

每个用户实例运行独立的文件服务：

- **端口**: 9876
- **路径**: `/files/*`
- **CORS**: 已启用

### API 端点

```bash
# 健康检查
GET http://服务器IP:9876/health

# 预览文件
GET http://服务器IP:9876/files/chart.png

# 下载文件
GET http://服务器IP:9876/files/report.pdf
```

## 注意事项

1. **文件路径格式**: 必须以 `/workspace/` 开头或使用相对路径 `./`
2. **文件服务**: 确保文件服务已启动并监听 9876 端口
3. **CORS**: 文件服务已配置 CORS，允许跨域访问
4. **安全性**: 生产环境建议启用 token 验证
